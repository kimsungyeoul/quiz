# 📘 **강의 관리 시스템 (Course Management System)**

> **유형**: 실습형 포트폴리오 제출 (소스코드 + 실행 결과 캡처)

---

## 1. 평가 개요

| 항목 | 내용                                                    |
| -- | ----------------------------------------------------- |
| 과목 | Java 프로그래밍                                      |
| 주제 | **강의 관리 시스템 (Course Management System)**              |
| 목표 | 클래스 설계, 상속/다형성, 배열 관리, 예외처리, `switch`, 정적 메서드, 기본 I/O |
| 제출 | ZIP(소스 전부) + 실행 결과 캡처(콘솔) + 출력 로그(txt)                |

---

## 2. 구현 목표

* 강의(Course) 정보를 **객체지향적으로 관리**하는 프로그램 작성
* **온라인 강의 / 오프라인 강의**는 출력 정보가 다르게 표시되어야 함
* **모든 강의는 배열**로 관리(`Course[]`, 고정 크기 5)
* **총 수강료 합계** 계산 및 **수강료 구간 라벨** 출력(`switch` 활용)

---

## 3. 클래스 구조 및 요구사항

| 클래스명            | 역할               | 비고          |
| --------------- | ---------------- | ----------- |
| `Course`        | 강의의 공통 속성과 기능 정의 | 부모 클래스      |
| `OnlineCourse`  | 온라인 강의           | `Course` 상속 |
| `OfflineCourse` | 오프라인 강의          | `Course` 상속 |
| `CourseManager` | 강의 등록/출력/합계      | 배열 사용       |
| `CourseTest`    | 실행 진입점 (`main`)  | 실행용         |

### (1) `Course` 클래스 — 모델 공통

* **필드**

  ```java
  protected String title;      // 과목명
  protected String instructor; // 강사명
  protected int fee;           // 수강료(원)
  ```
* **생성자**

  * 모든 필드 초기화 (유효성 검사 필수: null/blank 금지, 음수 금지)
* **메서드**

  * `void printInfo()` : 공통 정보 출력
  * `int getFee()` : 수강료 반환
  * (선택) `protected String baseInfo()` : 공통 문자열 포맷터

### (2) `OnlineCourse` 클래스 — 온라인

* **상속**: `Course`
* **추가 필드**: `int durationHours` (총 학습시간, 시간 단위, 1 이상)
* **오버라이딩**

  ```java
  @Override
  public void printInfo() {
      // 과목, 강사, 수강료, "온라인: X시간" 형식으로 출력
  }
  ```

### (3) `OfflineCourse` 클래스 — 오프라인

* **상속**: `Course`
* **추가 필드**: `String room` (강의실명, 공백 금지), `int seats` (좌석 수, 1 이상)
* **오버라이딩**

  ```java
  @Override
  public void printInfo() {
      // 과목, 강사, 수강료, "오프라인: R실 / 좌석: N" 형식으로 출력
  }
  ```

### (4) `CourseManager` 클래스 — 관리

* **필드**

  ```java
  private Course[] courses = new Course[5];
  private int count = 0;
  ```
* **메서드 (필수)**

  * `void addCourse(Course c)` : 배열에 추가 (용량 초과 시 예외 또는 안내 출력)
  * `void printAllCourses()` : 전체 목록 출력(다형성 호출) + **수강료 구간 라벨** 표시
  * `void printTotalFee()` : 총 수강료 합계 출력
* **정적 유틸(필수)**

  * `static String feeLabel(int fee)` : 다음 규칙에 따라 `switch`로 라벨 반환

    * 0만원대(저가), 1만원대, 2만원대, 3만원대, 4만원대, 5만원대 이상

### (5) `CourseTest` 클래스 — 실행

* **요구사항**

  * `CourseManager` 생성
  * `OnlineCourse`, `OfflineCourse` **3개 이상** 등록
  * (선택) `Scanner` 등으로 **추가 1개 입력 등록** 기능

    * 숫자 파싱 실패 시 `NumberFormatException` 처리
  * 전체 목록 출력 → 총 합계 출력

## 4. 실행 예시 (샘플)

```
추가로 오프라인 강의 1개 등록할까요? (Y/N): N
===== 강의 목록 =====
과목: 자바 기초 / 강사: 김교수 / 수강료: 18000원 / 오프라인: A101실 / 좌석: 30
 - 수강료 구간: 1만원대
과목: 파이썬 중급 / 강사: 이강사 / 수강료: 22000원 / 온라인: 24시간
 - 수강료 구간: 2만원대
과목: 클린 코드 워크숍 / 강사: 박멘토 / 수강료: 35000원 / 오프라인: B203실 / 좌석: 20
 - 수강료 구간: 3만원대
====================
총 수강료 합계: 75000원
```

> **주의**: 위는 예시일 뿐이며, 실제 과목/강사/수강료/출력 포맷은 요구사항만 지키면 자유.

## 7. 제약 & 권장사항

* **필수**: 표준 JDK만 사용 (외부 라이브러리 금지)
* **배열 고정**: `Course[]` 크기 **5** 고정(컬렉션 사용은 비교·연습용으로만 주석 처리)
* **예외 처리**: `NumberFormatException`, 값 검증 실패 시 사용자 메시지 표시
* **출력 포맷**: 예시 유사하게 가독성 좋게 정리
* **가시성**: 공통 필드 `protected`, 나머지 캡슐화 준수
