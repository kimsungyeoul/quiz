### 📍 **Part 3. 코드 퀴즈**

**16. 출력 결과**

```java
Bicycle constructor
MountainBike constructor
```

🧠 **설명:**
`MountainBike`의 생성자가 실행되기 전에 `super()`가 자동 호출되어 부모 클래스 `Bicycle`의 생성자가 먼저 실행됨.

---

**17. 컴파일 에러 이유**

> `SubClass`가 `SuperClass`를 상속받을 때, 부모 생성자 `SuperClass(String msg)`가 **기본 생성자(default constructor)** 를 제공하지 않기 때문에
> `SubClass` 생성자에서 명시적으로 `super("...")`를 호출해야 함.
> 하지만 코드에 `super()` 호출이 없어서 컴파일 에러 발생.

✅ 해결 예시:

```java
class SubClass extends SuperClass {
    public SubClass() {
        super("Hello");
    }
}
```

---

**18. 코드 완성**

```java
public class Car {
    private int speed;

    public Car(int speed) {
        this.speed = speed;
    }
}
```

🧠 생성자는 매개변수를 통해 speed 필드를 초기화해야 함.

---

**19. 출력 결과**

```
double
```

🧠 `3.14f`는 `float`형이지만 `float`에 대한 오버로딩이 없기 때문에 **가까운 타입인 double형 메서드**가 호출됨.

---

**20. 정답:** ✅ **A, C**

* `public class Hello {}` : O (파일명과 일치할 경우 public 가능)
* `class Hello {}` : O (default 접근제어자)
* `private`, `protected` 클래스는 **최상위(top-level)** 선언 불가
