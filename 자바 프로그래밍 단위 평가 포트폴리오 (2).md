# 🧩 **📘 자바 프로그래밍 단위평가 포트폴리오 문제**

---

## 🧾 **1. 평가 개요**

| 항목        | 내용                                 |
| --------- | ---------------------------------- |
| **평가 과목** | Java 프로그래밍                    |
| **평가 주제** | 도서 관리 시스템 (Book Management System) |
| **평가 목적** | 클래스 설계, 상속, 캡슐화, 다형성, 배열 활용 능력 평가  |
| **평가 형태** | 실습형 포트폴리오 제출 (소스코드 + 실행 결과 캡처)     |

---

## 🎯 **2. 구현 목표**

도서(Book) 정보를 관리할 수 있는 **객체지향 기반 프로그램**을 작성하시오.
책의 종류(전자책, 종이책)에 따라 출력 정보가 달라지며,
모든 책을 배열로 관리해야 한다.

---

## 🧱 **3. 클래스 구조 및 요구사항**

| 클래스명          | 역할                   | 비고        |
| ------------- | -------------------- | --------- |
| `Book`        | 도서의 공통 속성과 기능 정의     | 부모 클래스    |
| `EBook`       | 전자책 클래스              | `Book` 상속 |
| `PrintedBook` | 종이책 클래스              | `Book` 상속 |
| `BookManager` | 도서 등록 및 출력 관리        | 배열 사용     |
| `BookTest`    | 프로그램 실행 진입점 (`main`) | 실행용 클래스   |

---

### ✅ **(1) Book 클래스**

* **필드**

  ```java
  protected String title;
  protected String author;
  protected int price;
  ```
* **생성자**

  * 모든 필드를 초기화하는 생성자
* **메서드**

  * `void printInfo()` : 도서 정보 출력
  * `int getPrice()` : 가격 반환

---

### ✅ **(2) EBook 클래스**

* `Book` 클래스를 상속받는다.
* **추가 필드:** `int fileSize` (단위: MB)
* **메서드 오버라이딩**

  ```java
  @Override
  public void printInfo() {
      // 제목, 저자, 가격, 파일크기 출력
  }
  ```

---

### ✅ **(3) PrintedBook 클래스**

* `Book` 클래스를 상속받는다.
* **추가 필드:** `int pageCount`
* **메서드 오버라이딩**

  ```java
  @Override
  public void printInfo() {
      // 제목, 저자, 가격, 페이지 수 출력
  }
  ```

---

### ✅ **(4) BookManager 클래스**

* **필드**

  ```java
  private Book[] books = new Book[5];
  private int count = 0;
  ```
* **메서드**

  * `void addBook(Book b)` : 배열에 책 추가
  * `void printAllBooks()` : 전체 도서 목록 출력
  * `void printTotalPrice()` : 총 가격 합산 후 출력

---

### ✅ **(5) BookTest 클래스 (main 실행)**

* `BookManager` 객체 생성
* `EBook` 및 `PrintedBook` 객체 3개 이상 생성
* 등록 및 출력 메서드 호출

```java
public class BookTest {
    public static void main(String[] args) {
        BookManager manager = new BookManager();

        manager.addBook(new PrintedBook("자바의 정석", "남궁성", 30000, 1200));
        manager.addBook(new EBook("파이썬 완벽 가이드", "권철민", 25000, 15));
        manager.addBook(new PrintedBook("Clean Code", "Robert C. Martin", 35000, 550));

        manager.printAllBooks();
        manager.printTotalPrice();
    }
}
```
### ✅ **(6)
* static 메서드를 만드는 방법은 제한이 없는데, 추천한다면 Helper 클래스 하나 만들고 특정 책이 전자책인지 종이책인지를 판별해 주는 static 메서드를 만들면 좋을 것 같음.
* 이 static 메서드에서 switch 구문을 사용하는 것을 추천
* Enum은 정의하면 좋을 것 같음. 대신에 public static final 상수로 전자책, 종이책을 정의해도 됨
